# カキフォイの日記 — 運用ルール

## カテゴリ（固定リスト）

記事の `category` は必ず以下の7つから選択する。**新規カテゴリは作らない。**
組み合わせカテゴリ（「グルメ・日常」のような表記ゆれ）も作らない。

- グルメ
- 旅行・おでかけ
- 子育て・日常
- ゲーム・エンタメ
- ガジェット・テクノロジー
- お金・投資
- 健康・ランニング

迷ったら記事の主題（読者が何を求めて読むか）で判断する。
例：保育参観のあとのランチ記事 → 主題が店の紹介なら「グルメ」。

細かい切り口（習い事、節約、美容など）はカテゴリではなくタグで表現する。

## タグ

- 新しいタグを付ける前に既存タグを確認し、表記ゆれを作らない
  （例：×「牛たん」→ ○「牛タン」、×「Nightreign」→ ○「ナイトレイン」）
- 英語表記のタグは原則作らない（固有名詞のサービス名・製品名は除く: Discord, iPhone等）
- 地名は市単位を基本にする（籠原→熊谷、深谷市→深谷）

## 記事ファイル

- 記事: `src/content/blog/[slug].md`
- 画像: `public/images/`（記事内パスは `/images/xxx.jpg`）
- アイキャッチ: 1200×675px、`heroImage` にpublicパス文字列で指定
- 商品カードのCSSは `global.css` に集約済み

## デプロイ

- Astro + Cloudflare Pages。`git push origin main` で自動デプロイ
- 旧カテゴリURLの301リダイレクトは `public/_redirects` で管理
